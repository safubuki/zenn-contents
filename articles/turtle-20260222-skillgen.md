AIのカスタマイズ機能を利用して、AIを作るためのAIを構築する……この「メタ生成」のアイデアは非常に強力です。
そして昨今、Anthropicの「Claude Code」によって提唱・規格化され、GitHub Copilotなどの最新AIツールにも採用されているカスタム指示の仕組みが広がっています。それが「**Agent Skills**」です。

Agent SkillsはこれまでのGitHub CopilotのCustom Instructions（カスタム指示）などとは、異なる特徴を持っています。その最大の違いが「発動条件（トリガー）」の概念です。さらに、プロジェクトのディレクトリ構造全体を読み込ませたり、外部ファイルを参照させたり、スクリプトを実行することも可能です。

本記事では、このAgent Skillsの利点から、あなただけのスキルを量産するための強力なメタスキル「**Skills Generator**」の全貌と完全な作り方までを公開します。

## Custom Instructions と「Agent Skills」の決定的な違い

これまでにも、AIエディタには「AIに前提ルールを教え込む」機能が存在しました。代表的なものがGitHub Copilotの **Custom Instructions**（`.github/copilot-instructions.md`等）です。さらに、特定のリポジトリやディレクトリ向けに個別の `instructions.md` や `rules.md` を作成している方も多いでしょう。

では、これらの既存機能と「**Agent Skills**」は何が違うのでしょうか？
一言で表すなら、**「常にかかるルール（憲法）」か「必要な時だけ呼ばれる専門スキル（マニュアル＋道具箱）」か**の違いです。

GitHub Copilot公式も推奨している、実際の開発現場におけるスマートな使い分けは以下のようになります。

### 1. Custom Instructions / rules.md （憲法・共通ルール）
* **役割**: どう振る舞うか（行動方針・制約・書き方のルール）を指定する。
* **使われ方**: コーディング時に**常時**（または対象パス内で常に）自動適用される。
* **向いている内容（常に守ってほしい基本方針）**:
  * 「TypeScriptのstrictモードを維持して」
  * 「出力形式は箇条書きを優先し、コードには日本語コメントをつける」
  * 「常にセキュリティを最優先して実装方針を出力する」

### 2. Agent Skills （専門作業マニュアル＋道具箱）
* **役割**: 何をどう実行するか（特定タスク用の手順・ノウハウ・補助リソース）をパッケージ化する。
* **使われ方**: ユーザーのチャット内容（指示）に**関連する時だけ、AIがオンデマンドで自動選択**（または明示的に呼び出し）する。
* **向いている内容（特定の作業だけで使うフローや手順）**:
  * 「PRレビューをこの特定の観点・手順で回す」
  * 「Remotionを使った動画生成の構成を作り、自動化スクリプトを実行する」
  * 「ロボット制御ドライバのセーフガード検証スクリプトを回しながら修正する」

### コンテキストを汚さない「オンデマンドな能力拡張」

多様な作業（コーディング、テスト作成、動画自動生成など）を行うプロジェクトにおいて、すべての手順やノウハウを一つの `instructions.md` に詰め込むとどうなるでしょうか？
**AIがルール過多で混乱し、本当に必要な指示を見落とす**という致命的な弱点が生じます。動画生成コードを書きたいだけなのに「Web用のエラーハンドリング」が裏でずっと発動しているのは不自然ですし、AIのコンテキストウィンドウの無駄遣いにもなります。

かといって、タスクごとに専用の `.md` ファイルをたくさん作って管理し、手動でチャットのたびに「このタスクでは `@video-rules.md` を読み込んで」とAIに指定し続けるのも非常に手間です。

**Agent Skills** はこの二律背反を見事に解決します。AIがユーザーのチャット内容（「動画生成スクリプトを作って」「安全性を検証して」など）を見て、あらかじめ用意された大量のスキルパッケージの中から**「いま必要な専門スキル（SKILL.md）」だけを自動的に判断してコンテキストに注入**してくれます。

さらに、プロンプトテキストだけでなく、補助スクリプト（`scripts/`）や出力テンプレート（`templates/`）も一つのフォルダに同梱できるため、単なるルール文書を超えた強力な「能力モジュール」として機能するのです。

## なぜ「スキルを作るスキル（Skills Generator）」が必要なのか？

このように非常に強力でスマートなAgent Skillsですが、いざ自分で作ろうとすると大きな壁にぶつかります。
**作り方が難しい**のです。

Agent Skillsは単なるテキストボックスへの入力ではなく、プロジェクト内に `SKILL.md` というMarkdownファイルを作り、AIが発動条件を理解するための正確な `description`（説明文とキーワード）を書き、場合によっては複数の参照ファイル（`references/`）を連携させる必要があります。

「プロンプトのコツ」だけでなく「ディレクトリ構成のコツ」まで求められるため、「便利なのはわかるけど、自分で書くのは面倒くさい……」となってしまう方が大半でしょう。

Agent Skillsのポテンシャルは単なるプロンプト（テキスト）を遥かに超えます。
私が作成した `skills-generator` は、「1つの長いプロンプト」ではなく、**「専門知識のデータベース」を内包したエコシステム**です。

実際の `skills-generator` の仕組みは以下の図のようになっています。

```text:Skills Generatorの構造と役割
.agent/skills/skills-generator/
├── SKILL.md              # 【メイン指示書】4つのモード（作成・改善・監査・翻訳）を駆動
├── assets/               # 💎 出力用テンプレート集
│   └── skill-template.md # 完璧なSKILL.mdのひな型
├── references/           # 📚 専門知識とルールのデータベース
│   ├── best-practices.md # Copilotが確実に発火するためのルール集
│   ├── security-checklist.md
│   └── script-patterns.md
└── examples/             # ✨ 理想的なスキルの完成例
    └── example-skill/SKILL.md
```

AIエージェントが「新しいスキルを作って」という指示を受けたとき、単に推測で書き始めるわけではありません。**これらのフォルダ内の知識（ベストプラクティスやテンプレート）をすべて読み込み、プロの「スキルアーキテクト」として動作します。**
だからこそ、目的がどれだけ複雑であっても、決して破綻しない高品質なスキルパッケージを生み出せるのです。

## 『Skills Generator』の完全な作成手順

それでは、実際に皆さんの環境にこの `skills-generator` を構築する手順を解説します。
以下の内容をコピーペーストしてファイルを作成するだけで、誰でもすぐに使い始めることができます。

### Step 1: 環境別セットアップ（保存場所の準備）

Agent Skillsは、使用するAIエディタやツールによって読み込まれるディレクトリが異なります。ご自身の環境に合わせて、プロジェクトのルートディレクトリに以下のフォルダを作成してください。

#### A. GitHub Copilot
- **利用ディレクトリ**: `.github/skills`
- **設定**: VS Code等の `.vscode/settings.json` で以下を有効にします。
```json
{
  "chat.useAgentSkills": true
}
```

#### B. GPT Codex (CLI)
- **利用ディレクトリ**: `.agents/skills`
- **設定**: **不要**。プロジェクトルートに `.agents/skills` ディレクトリが存在するだけで自動的に認識されます。

#### C. Google Gemini (AntiGravity)
- **利用ディレクトリ**: `.agent/skills`
- **設定**: **不要**。プロジェクトルートに `.agent/skills` ディレクトリが存在するだけで自動的に認識されます。

### Step 2: フォルダ構造の作成

作成した `skills` ディレクトリの直下に、以下のように `skills-generator` フォルダとその中身（`assets` など）を作成していきます。（※以降の解説では、パス表記の例として AntiGravity の `.agent/skills` を使用します。ご自身の環境に合わせて読み替えてください）

```text:ディレクトリ作成
.agent/skills/skills-generator/
├── SKILL.md
├── assets/
│   └── skill-template.md
├── references/
│   └── best-practices.md
└── examples/
    └── example-skill/
        └── SKILL.md
```

### Step 3: メインの指示書（`SKILL.md`）の作成

次に、ジェネレーターの「脳」となるメインファイルを作成します。以下の内容をコピーして `.agent/skills/skills-generator/SKILL.md` に保存してください。

```markdown
# Skills Generator — Agent Skills のメタスキル

## スキル読み込み通知
このスキルが読み込まれたら、必ず以下の通知をユーザーに表示してください：
> 💡 **Skills Generator スキルを読み込みました**  
> Agent Skillsの作成・改善・診断・ローカライズを行います。

Agent Skills を作成・改善・診断・ローカライズするためのメタスキルです。
ユーザーの要望に応じて、以下の4つのモードで動作します。

| モード | 説明 | トリガー例 |
|--------|------|-----------|
| 🔨 **Generate** | 新しいスキルを一式作成 | 「pytestのスキルを作って」 |
| ✨ **Improve** | 既存スキルをベストプラクティスに基づき改善 | 「スキルを改善して」 |
| 🛡️ **Audit** | 既存スキルのセキュリティ診断 | 「スキルをチェックして」 |
| 🌐 **Localize** | 英語スキルの日本語化 | 「スキルを日本語化して」 |

---

## When to Use
- 新しい Agent Skill を作成したいとき
- 「こんなことできたらいいな」を実現するスキルが欲しいとき
- ヘルパースクリプト付きの自動化スキルを作りたいとき
- 既存スキルの発火条件・構造・品質を改善したいとき

## Mode 1: Generate — スキル新規作成

### Step 1: ヒアリング
ユーザーから以下の情報を確認する：
1. **目的**: 何をするスキルか
2. **スコープ**: プロジェクト固有か、汎用か
3. **自動化**: ヘルパースクリプトで自動化したい処理があるか

### Step 2: スキル構成の設計
以下のディレクトリ構成を基本とし、必要に応じてオプションフォルダを追加する：
- `scripts/`: 自動化スクリプトが必要な場合
- `references/`: 仕様書・チェックリストが必要な場合
- `examples/`: 実装例が必要な場合
- `assets/`: テンプレートが必要な場合

### Step 3: SKILL.md の作成
SKILL.md テンプレートは以下を参照：
📄 **[assets/skill-template.md](assets/skill-template.md)**

作成時のルール：
1. **name**: 小文字の英字・数字・ハイフンのみ（最大64文字）
2. **description**: 1行で完結させる。目的と発火キーワードを必ず含める。
3. **通知**: スキル読み込み通知を含める。
4. **構造**: 詳細なリストやコードは外部ファイルに分離する。

### スキル設計のベストプラクティス
詳細は必ず以下を参照：
📄 **[references/best-practices.md](references/best-practices.md)**

## Mode 2: Improve — 既存スキルの改善

既存スキルを分析し、発火条件の最適化・構造改善を行います。
- SKILL.md が肥大化している場合は外部ファイルに分離する。
- ユーザーが使いそうな発火キーワードを `description` に追加する。

---

## Mode 3: Audit / Mode 4: Localize
- 対象スキルのリスク（不可逆な削除、外部へのデータ送信等）を警告する。
- 英語スキルの場合、コードブロックやコマンドを破壊せずに日本語化する。
```

### Step 5: ルールブック（`references/best-practices.md`）の作成

ここでAIに「スキルの名前の付け方」や「確実に発火させるコツ」を教え込みます。`.agent/skills/skills-generator/references/best-practices.md` に保存します。

:::details 【コピペ用】best-practices.md（クリックして開く）
```markdown
# スキル設計のベストプラクティス

## 1. description の書き方（最重要）

`description` はエディタのAIがスキルを読み込むかどうかを判断する唯一の材料です。
- 1行で完結していること（改行を使わない）
- 「何をするスキルか」「どんな時に使うか」が明記されていること
- 発火キーワードが具体的に列挙されていること

## 2. SKILL.md の構成原則

SKILL.md は「手順書」であり「百科事典」ではない。
- **手順（Step）と方針** → SKILL.md に記述
- **テンプレート・ひな型** → `assets/` に分離
- **詳細な仕様・リファレンス** → `references/` に分離
- **コード例・サンプル** → `examples/` に分離

## 3. name の命名規則

- 小文字英字 `a-z`、数字 `0-9`、ハイフン `-`のみ（最大64文字）。
- 具体的な機能を端的に表す名前にする（例: `unit-testing`, `security-audit`）。

## 4. フォルダの追加条件

SKILL.md が50行を超えるような長いテンプレートやリファレンスを持つ場合、必ず外部ファイルに分離し、相対パスリンクで参照する。
```
:::

```markdown
## 手順

命名、構造、DRY原則、エラーハンドリングなどの観点でチェックを行う。

### Step 3: レポート出力
📄 **[assets/review-template.md](assets/review-template.md)** のテンプレートに従って出力。
```

## 実際にスキルをジェネレートしてみる

作成した `skills-generator` の威力を試してみましょう。
エディタのチャットを開き、以下のように依頼します。

```text:プロンプト
Zenn用のブログ記事を一貫したトーンで書けるように、私の過去記事を分析して人間らしい文章を生成する「執筆スキル」を作って。AIっぽさは極力排除してほしい。
```

すると、AIはただの文章ではなく、私たちが先ほど用意した `best-practices.md` や `skill-template.md` を裏で読み込み、以下のような**完璧に構造化されたスキルセット**を一瞬で構築・提案してくれます。

![スキル生成例のイメージ画像](/images/turtle-20260222-skillgen/make-skill-example.png =600x)

:::details AIが自動生成したスキルの役割定義（例：zenn-article-generator）
```markdown
## 役割と目標 (Role and Goal)
* ユーザーから提供されたブログテーマを最優先とし、`articles` ディレクトリ内の既存の記事（参照ブログ）を深く分析することで、筆者本人の「人間らしい」文体・癖・リズムを完全に再現したブログ記事を作成する。
* 「AIが書いたような無難で単調な言い回し」を徹底的に排除する。

## 行動ルール (Actions and Rules)
### 1. 参照ブログの深層分析
表面的な文体だけでなく「癖」を抽出する。
* 文体・トーン: 一人称のブレ、読者への語りかけ方、熱量の乗せ方。
* リズム: 短文の連続によるテンポ感、体言止めの活用、余白の使い方。

### 2. AI特有の表現の徹底的な排除 (Anti-AI Patterns)
以下の「AI構文」を**絶対に**使用してはならない。
* ❌ 禁止表現
  * 「〜は重要です」「〜は不可欠です」
  * 「結論として、」「要約すると、」「まとめとして、」
```
:::

人間が書くと抜け漏れが発生しがちな「エラー処理」や「AIへの禁止事項」まで、プロのアーキテクトとしてガードレールを組み込んでくれるのが、このジェネレーター最大のメリットです。

## 発展的な領域：複数スキルの連携（エコシステム）

このようにして量産したAgent Skillsは、単一のコード生成に留まりません。
複数のスキルを役割ごとに「チェーン（連動）」させ、ひとつの巨大な「処理パイプライン」を形成することも可能です。

以下の図は、Reactを使って動画をプログラマブルに生成するフレームワーク「**Remotion**」を活用し、動画制作を完全自動化するための「**Remo Skill Ecosystem Map**」の概念図です。

![Remo Skill Ecosystem Map](/images/turtle-20260222-skillgen/remo-ecosystem.png =600x)

このエコシステムでは、ひとつの強大なAIにすべてをやらせるのではなく、役割を細分化した複数のAgent Skillsが数珠つなぎに連携します。

1. **User** が「こんな動画を作りたい」と指示。
2. 総監督である **professional-video-director** スキルが受け取り、全体の構成と方針を策定。
3. 画像生成プロンプトを作るスキル（`remo-image-prompt-gen`）、台本を書くスキル（`remo-script-writer`）にタスクを分割して並列処理。
4. **remo-composition** スキルがRemotionのコードとして組み上げ、品質チェックを経てMP4としてレンダリング。

このように単機能の「職人スキル」を大量に生み出し、AIエージェントを「チーム」として自律動作させることが可能になります。
これほど多くのカスタムスキルを量産するにあたっても、大元となる「**Skills Generator**」の存在が不可欠な基盤となっています。
