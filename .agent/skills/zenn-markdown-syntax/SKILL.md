# Zenn Markdown Syntax (記法リファレンス管理)

## スキル読み込み通知

このスキルが読み込まれたら、必ず以下の通知をユーザーに表示してください：

> 💡 **Zenn Markdown Syntax スキルを読み込みました**
> Zenn独自のMarkdown記法（メッセージブロック、埋め込み仕様など）のリファレンスを提供し、正しい文法で出力します。

---

## 役割と目標 (Role and Goal)

* ブログ記事を作成・編集する際、Zenn特有の拡張されたMarkdown記法（メッセージ、トグル、コンテンツ埋め込み、数式、画像のリサイズなど）に関する知識を提供し、正しい記述を保証するスキルです。

## リファレンスの活用

ZennのMarkdown記法の詳細、書き方の具体例、注意事項については以下の参照ファイルを参照してください：

📄 **[references/zenn-markdown-guide.md](references/zenn-markdown-guide.md)**

記事の本文を構築する際、あるいはユーザーからZenn記法に関する質問があった際は、必ずこのリファレンスの内容を読み込んでから回答または記述を行ってください。

### 主なZenn独自の記述 (Actions and Rules)

以下は特に頻出する独自の記法ルールです。詳細はリファレンスを参照してください。

| 機能 | 記法例 | 補足 |
| ---- | ------ | ---- |
| **メッセージ** | `:::message`<br>（メッセージ内容）<br>`:::` | 警告は `:::message alert` |
| **トグル（アコーディオン）** | `:::details タイトル`<br>（表示内容）<br>`:::` | ネストする場合は `::::details` と `:` を増やす |
| **リンクカード** | `@[card](URL)` | または単独の行にURLのみ記述 |
| **画像の横幅指定** | `![](URL =250x)` | ピクセル数で幅を指定可能 |
| **ファイル名つきコード** | \`\`\`js:ファイル名 | |
| **数式ブロック** | `$$`<br>（数式）<br>`$$` | KaTeX対応。前後に空行が必要 |

---

## 留意事項
* 共起発火: `zenn-article-generator` やその他のスキルにてブログ記事が新規作成される際、ユーザーが提示した内容に「コードブロック」「注意書き」「リンクカード」などの要素が相応しいと判断された場合は、本スキルの `references` に従い、Zenn特有の記法を用いて出力してください。
